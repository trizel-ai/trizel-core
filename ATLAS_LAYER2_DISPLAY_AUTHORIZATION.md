## 8. Zenodo Metadata Display Protocol

### 8.1 Authorized Zenodo Metadata Elements

For Zenodo records related to the ATLAS / 3I program, Layer-2 may display:

1. **Bibliographic Metadata:**
   - Record title (as registered in Zenodo)
   - DOI (persistent identifier)
   - Authors/Contributors (names and affiliations)
   - Publication/Deposit date
   - Record version (if applicable)

2. **Access Information:**
   - Access status (OPEN, EMBARGOED, CLOSED, RESTRICTED)
   - Embargo end date (if applicable and publicly visible)
   - License type (if publicly visible)
   - Access conditions statement (from Zenodo public metadata)

3. **Classification Metadata:**
   - Record type (dataset, publication, software, etc.)
   - Keywords or tags (if publicly visible in Zenodo)
   - Related identifiers (if publicly visible)
   - Communities (if record is part of Zenodo communities)

### 8.2 Prohibited Zenodo Content

Layer-2 **may NOT display**:

- Contents of files within the record
- Data previews or samples
- Abstracts or descriptions from embargoed/closed records (unless visible in Zenodo's public interface)
- Download statistics or access metrics
- User comments or annotations
- Any content requiring authentication or special access

### 8.3 Zenodo Display Verification

Before displaying Zenodo metadata:

1. **Verify Public Availability:** Confirm metadata is publicly visible on Zenodo.org
2. **Record DOI:** Document the specific Zenodo record DOI
3. **Screenshot or Archive:** Maintain record of Zenodo public metadata state
4. **Governance Review:** Submit display content for governance compliance check
5. **Update Protocol:** Establish protocol for updating if Zenodo record status changes

### 8.4 Linking Protocol

- Direct links to Zenodo record landing pages are **PERMITTED**
- Links must point to Zenodo.org canonical URLs
- Links must not bypass Zenodo access controls
- Links must not link directly to files within closed/embargoed records
